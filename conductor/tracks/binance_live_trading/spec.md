# Binance Live Trading - Specification

## Objective
Transition the automated Binance trading model from simulation (dry run) to live trading once the backtested model achieves maturity, stability, and consistent profitability (e.g., >10% - 20% consistently).

## Scope
- Monitor the daily morning summaries to evaluate the auto-researcher's recommendations and profitability metrics.
- Develop a `live_trade.py` execution script that interacts with Binance's real order endpoints via the MCP server or direct API calls.
- Implement strict risk management rules (e.g., maximum daily loss, hard stop losses, position sizing limits) to protect capital during the initial live phase.
- Create a twin project structure in `projects/binance_live_trading/` that inherits the winning `strategy.py` from the simulation track but executes trades with real funds.
- Expand Telegram notifications to report real-time live execution logs, filled orders, and daily PnL in live conditions.

## Out of Scope
- Developing entirely new trading algorithms from scratch (live trading relies on the winning strategies generated by the existing simulation auto-researcher).
- High-frequency trading (HFT) infrastructure.
